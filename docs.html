BaKangombe API Documentation

Overview

BaKangombe provides realistic fake data for prototyping, testing, and development. No API key is required for the free tier. The API returns JSON responses with randomly generated data each time you call an endpoint.

Base URL

https://ba-kangombe.vercel.app

Authentication

Free tier: No authentication required. Simply make requests to the endpoints.

Premium tier: Requires an API key obtained after subscribing. Include the key in the x-api-key header.

curl -H "x-api-key: YOUR_API_KEY" https://ba-kangombe.vercel.app/api/users/5

Premium keys grant 10,000 requests per minute, custom schemas, CSV export, and priority support.

Rate Limits

PlanRequests per minute
Free100
Premium10,000

If you exceed your limit, the API returns a 429 error.

Endpoints

GET /api/users/:count?

Returns an array of fake user objects. Optional count parameter (1‑100). Default 5.

Example: /api/users/2

{
  "count": 2,
  "data": [
    {
      "id": "uuid",
      "firstName": "John",
      "lastName": "Doe",
      "email": "john@example.com",
      "avatar": "https://...",
      "phone": "123-456-7890",
      "address": { "street": "...", "city": "...", "state": "...", "zipCode": "...", "country": "..." },
      "jobTitle": "Engineer",
      "company": "Acme Inc",
      "createdAt": "2025-01-01T00:00:00.000Z"
    }
  ]
}

GET /api/products/:count?

Returns fake product objects (name, price, rating, etc.). Optional count (1‑100, default 10).

{
  "count": 1,
  "data": [{ "id": "...", "name": "Product", "price": 9.99, "category": "Electronics", ... }]
}

GET /api/company

Returns a single fake company profile with CEO, revenue, headquarters, etc.

{
  "id": "...",
  "name": "TechCorp",
  "catchPhrase": "Innovative solutions",
  "industry": "Software",
  "employees": 450,
  "founded": 2010,
  "headquarters": { ... },
  "ceo": "Jane Smith",
  "revenue": "$85B",
  "contactEmail": "contact@techcorp.com",
  "contactPhone": "+1 555 123 4567"
}

GET /api/transactions/:count?

Returns fake payment transactions (amount, status, payment method, etc.). Optional count (1‑100, default 20).

{
  "count": 2,
  "data": [
    { "id": "...", "amount": 99.99, "currency": "USD", "status": "completed", ... }
  ]
}

Error Codes

Status CodeMeaning
400Bad request (invalid count, etc.)
401Unauthorized – invalid API key (premium tier)
429Rate limit exceeded

Code Examples

cURL

curl https://ba-kangombe.vercel.app/api/users/3

JavaScript (fetch)

fetch('https://ba-kangombe.vercel.app/api/users/5')
  .then(res => res.json())
  .then(data => console.log(data));

Python (requests)

import requests
response = requests.get('https://ba-kangombe.vercel.app/api/company')
print(response.json())
